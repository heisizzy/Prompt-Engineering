1 — Introduction

This module trains readers to become prompt engineers: people who can design, test, and optimize prompts to get predictable, high-quality results from AI systems across text, code, and images. The lessons are hands-on with templates, examples, and visual diagrams.

2 — Prompt anatomy (the basics)

Core parts: system message (where supported), instructions, context (documents, memory), examples (few-shot), and output format. Treat prompts like software: break them into components and test each.

Prompt recipe (simple)

1) Goal: What I want.\n2) Context: any facts, documents.\n3) Constraints: length, style, format.\n4) Examples: 1–3 examples if helpful.\n5) Output spec: JSON, bullets, or plain text.
Exercise: Take this instruction: "Summarise latest research on X" and rewrite it using the prompt recipe above.

3 — Beginner: practical patterns & templates

Start simple. Beginners should practice templates that are reproducible. Copy → run → adjust.

1) Instruction + Output format

Write a one-paragraph summary of the article below in <=80 words.\nArticle: [PASTE TEXT]\nSummary:

2) Role-play template

You are an expert UX researcher. Provide 5 usability issues found in the following app description, with numbered bullets and short fixes.\nApp: [PASTE]

3) Few-shot for style control

Example Input: ...\nExample Output: ...\nNow follow the pattern above for this input: [NEW INPUT]
Secret (beginner): When a model is vague, add a tiny constraint: e.g., "Answer in 3 sentences, each under 20 words." Micro-constraints steer models strongly.

4 — Intermediate: system prompts, few-shot, chaining

This is where prompt engineering becomes a workflow. Use system messages to set identity & guardrails; few-shot to teach patterns; chaining to split tasks.

System prompt examples

System: You are a careful, factual assistant. If unsure, say "I don't know" and ask to look it up. Always show reasoning steps when asked.

Chain-of-thought & decomposition

Break complex tasks into smaller steps and use the model to solve each step. Example:

1) Extract facts from document.\n2) Generate outline.\n3) Draft each section.\n4) Combine and polish.
Exercise: Turn the prompt "Plan a 5-day trip to Cape Town" into a 4-step chain: (1) gather constraints, (2) suggest itinerary, (3) budget, (4) final itinerary.

5 — Pro: RAG, tools, multimodal, evaluation

At the pro level you combine models with external tools and retrieval. The goal: reliable, up-to-date, and auditable outputs.

Retrieval-Augmented Generation (RAG)

Pattern: retrieve relevant documents → condense → ask LLM to answer grounded on those passages. Always include citations to document IDs or short snippets.

Step 1: Retrieve top-k docs for query.\nStep 2: Extract relevant paragraphs (with loc IDs).\nStep 3: Provide prompt: \"Answer using ONLY the paragraphs below. Cite paragraph IDs.\"

Tool use & instruction

Many systems let you call tools (code execution, calculators, web search). Design the prompt to identify when to call the tool and what to do with results. Keep tool responses small and machine-friendly (JSON) so the LLM can continue processing.

Multimodal prompting

Combine text + images: provide short captions, bounding boxes, or alt-text. For images, always include the question and specify what to output—e.g., object list, caption, or JSON of coordinates.

Secret (pro): Use "sparse grounding" — feed only the most densely relevant passages (2–5) instead of everything. It reduces hallucination and speeds inference.

6 — Workflow: test, measure, iterate

Treat prompts like experiments. Track: input variants, model settings (temp, max tokens), outputs, and evaluation scores. Keep a CI-style prompt suite for important prompts.

Evaluation metrics

  • Accuracy: factual correctness when applicable
  • Relevance: how well the answer matches the request
  • Conciseness: brevity and structure
  • Robustness: performance under small input changes

Testing checklist

• Run N=10 varied inputs.\n• Check for hallucinations.\n• Confirm format validity (JSON parse).\n• Measure consistency (semantic similarity between outputs).

7 — Secrets & advanced heuristics

These are high-leverage tricks used by experienced prompt engineers.

  1. Signal tokens: Include tokens like "Answer concisely:" or "Step-by-step:" — models learn these anchors quickly.
  2. Delimiter trick: Use triple backticks or XML tags to separate context so models don't conflate parts.
  3. Round-trip checking: Ask the model to generate an answer and then ask it to verify or critique its own answer.
  4. Temperature staging: Use higher temp for ideation, low temp for finalization. Combine outputs (ensemble).
  5. Bias probes: Test with adversarial inputs to reveal unsafe or biased completions.
Pro secret: "Debias then refine." First ask the model to produce diverse candidate answers from deliberately opposite viewpoints (high temp). Then use a low-temp evaluator prompt that ranks or picks the least-biased, most-factual candidate.

8 — Anti-patterns & what to avoid

  • Overloading a single prompt with multiple unrelated tasks — split into chain steps.
  • Relying on implicit knowledge without grounding — always cite when facts matter.
  • Using too many examples — 1–3 clear examples outperform many low-quality ones.
  • Hard-coding the expected answer into the prompt examples (leaks).

9 — Ethics, safety & deployment checklist

Design prompts with guardrails. When deploying, include monitoring for hallucinations, toxicity, and PII leaks.

• Add explicit refusal style: "If asked for illegal info, refuse politely."\n• Monitor outputs for PII.\n• Keep human-in-the-loop for critical decisions.

12 — Quick reference cheat-sheets

Temperature & creativity

  • 0.0–0.3: deterministic, factual
  • 0.4–0.7: creative, controlled
  • 0.8–1.0: high creativity, variation

Token budget tips

  1. Shorten context: keep only the most relevant passages.
  2. Compress with summaries before asking for synthesis.
